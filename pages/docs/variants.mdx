# Variants

Variants allows you to create multiple versions of the same component. 


### Adding variants

You can add variants by using the `variants` key. There's no limit to how many variants you can add.
<br/>

<button class="font-semibold text-white text-sm py-1 px-4 rounded-full active:opacity-80 bg-purple-500 hover:bg-purple-700">
  Button
</button>

```js {5-11, 14}
import { tv } from 'tailwind-variants';

const button = tv({
  base: 'font-semibold text-white text-sm py-1 px-4 rounded-full active:opacity-80',
  variants: {
    color: {
      primary: "bg-blue-500 hover:bg-blue-700" ,
      secondary: "bg-purple-500 hover:bg-purple-700",
      success: "bg-green-500 hover:bg-green-700",
    },
  },
})

button({ color: "secondary" })
/**
 * font-semibold text-white text-sm py-1 px-4 rounded-full active:opacity-80 bg-purple-500 
 * hover:bg-purple-700
 */
```

<Callout type="info" >
  <div class="text-sm">
    You can also use the values as an array. e.g.&nbsp;
    <code>base: ['font-semibold', 'text-white', ...]</code>
  </div>
</Callout>

#### Multiple variants

You can add multiple variants to a single component.

<br/>

<div class="flex items-center flex-start gap-3">
  <button class="font-semibold text-white rounded-full active:opacity-80 bg-blue-500 hover:bg-blue-700 py-1 px-3 text-xs">
    Primary
  </button>
  <button class="font-semibold text-white rounded-full active:opacity-80 bg-purple-500 hover:bg-purple-700 py-1.5 px-4 text-sm">
    Secondary
  </button>
  <button class="font-semibold text-white rounded-full active:opacity-80 bg-green-500 hover:bg-green-700 py-2 px-6 text-md">
    Success
  </button>
</div>

```js {5-16, 19, 25, 31}
import { tv } from 'tailwind-variants';

const button = tv({
  base: 'font-semibold text-white py-1 px-3 rounded-full active:opacity-80',
  variants: {
    color: {
      primary: "bg-blue-500 hover:bg-blue-700" ,
      secondary: "bg-purple-500 hover:bg-purple-700",
      success: "bg-green-500 hover:bg-green-700",
    },
    size: {
      sm: "py-1 px-3 text-sm",
      md: "py-1.5 px-4 text-md",
      lg: "py-2 px-6 text-lg"
    },
  },
})

button({ color: "primary", size: "sm" })
/**
 * font-semibold text-white rounded-full active:opacity-80 bg-blue-500 hover:bg-blue-700 
 * py-1 px-3 text-xs
 */

button({ color: "secondary", size: "md" })
/**
 * font-semibold text-white rounded-full active:opacity-80 bg-purple-500 hover:bg-purple-700 
 * py-1.5 px-4 text-sm
 */

button({ color: "success", size: "lg" })
/**
 * font-semibold text-white rounded-full active:opacity-80 bg-green-500 hover:bg-green-700 
 * py-2 px-6 text-md
 */
```


### Boolean variants

You can also add boolean variants to a component. This is useful when you want to add a state variant e.g. `disabled`.

<br/>

<button class="font-semibold text-white text-sm py-1 px-4 rounded-full active:opacity-80 hover:bg-purple-700 bg-gray-500 opacity-50 pointer-events-none">
  Disabled
</button>

```js {11-13, 17}
import { tv } from 'tailwind-variants';

const button = tv({
  base: 'font-semibold text-white text-sm py-1 px-4 rounded-full active:opacity-80',
  variants: {
    color: {
      primary: "bg-blue-500 hover:bg-blue-700" ,
      secondary: "bg-purple-500 hover:bg-purple-700",
      success: "bg-green-500 hover:bg-green-700",
    },
    disabled: {
      true: "opacity-50 bg-gray-500 pointer-events-none"
    }
  },
})

button({ color: "secondary", disabled: true })
/**
 * font-semibold text-white text-sm py-1 px-4 rounded-full active:opacity-80 
 * hover:bg-purple-700 bg-gray-500 opacity-50 cursor-not-allowed pointer-events-none
 */
```


### Compound variants

Sometimes you might want to add a variant that depends on another variant. For example, you might want to add a `color` variant that depends on the `disabled` variant. This is possible by using the `compoundVariants` key.


<br/>

<button class="font-semibold text-sm py-1 px-4 rounded-full active:opacity-80 hover:bg-green-700 opacity-50 pointer-events-none bg-green-100 text-green-600">
  Disabled
</button>

```js {15-21, 24}
import { tv } from 'tailwind-variants';

const button = tv({
  base: 'font-semibold text-white text-sm py-1 px-4 rounded-full active:opacity-80',
  variants: {
    color: {
      primary: "bg-blue-500 hover:bg-blue-700" ,
      secondary: "bg-purple-500 hover:bg-purple-700",
      success: "bg-green-500 hover:bg-green-700",
    },
    disabled: {
      true: "opacity-50 bg-gray-500 pointer-events-none"
    }
  },
  compoundVariants: [
    {
      color: "success",
      disabled: true,
      class: "bg-green-100 text-green-600" // You could also use "className" instead of "class"
    }
  ]
})

button({ color: "success", disabled: true })
/**
 * font-semibold text-sm py-1 px-4 rounded-full active:opacity-80 hover:bg-green-700 
 * opacity-50 pointer-events-none bg-green-100 text-green-600
 */
```

<Callout type="warning" emoji="⚠️">
  <div class="text-sm">
    Notes that the <code>compoundVariants</code> key is an array.
  </div>
</Callout>


### Default variants

You can also add a default variant to a component. This is useful when you want to add a predefined variants values to a component.

<br/>

<button class="font-semibold text-white rounded-full active:opacity-80 bg-blue-500 hover:bg-blue-700 py-1 px-3 text-sm">
Default
</button>

```js {17-20, 23}
import { tv } from 'tailwind-variants';
 
const button = tv({
  base: 'font-semibold text-white py-1 px-3 rounded-full active:opacity-80',
  variants: {
    color: {
      primary: "bg-blue-500 hover:bg-blue-700" ,
      secondary: "bg-purple-500 hover:bg-purple-700",
      success: "bg-green-500 hover:bg-green-700",
    },
    size: {
      sm: "py-1 px-3 text-sm",
      md: "py-1.5 px-4 text-md",
      lg: "py-2 px-6 text-lg"
    },
  },
  defaultVariants: {
    color: "primary",
    size: "md"
  }
})
 
button()
/**
 * font-semibold text-white rounded-full active:opacity-80 bg-blue-500 hover:bg-blue-700 
 * py-1 px-3 text-sm
 */
```
